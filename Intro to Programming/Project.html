Programming Basics: Programming and the Web

Lesson 1: The Basics of the Web

How does it work?

The "Web", or Internet, is essentially computers with browsers (Firefox, Chrome, Internet Explorer, etc.) connecting to other
computers, or servers, via HTTP requests. When you navigate to a webpage, you send a request from your browser to a
server in the form of an HTTP request and an HTML document is returned to your browser. This video shows the most basic
components of the Web, and this comment on an ELI5 (explain like I'm 5) question on reddit does a great job of explaining
how the internet works in more detail.

HTML

HTML is (or Hypertext Markup Language), for all intents and purposes, the basis for almost everything on the internet.
HTML documents are made up of hypertext (what you see) and markup (what it looks like). Udacity has a great video
that shows a basic overview here.

Elements of HTML

HTML documents are created by using tags. Tags are used to tell the browser which elements to display. Another udacity.com video
shows different tags and their purpose.

Block and Inline Elements

Two types of HTML elements, block and inline, differ in the way they handle content. Block elements create an invisible box,
or container, around the content while inline elements do not.

HTML Document Structure

A complete HTML document is made up of different elements put together and contain:
head - where the title, metadata, CSS and Java Script references, etc. are located
body - where all displayed content (text, images, videos, etc.) is located

Why We Code

We use code such as HTML because computers are stupid, meaning that they cannot think for themselves; they need to be
told, step-by-step, in their own language , exactly what we need them to do for us. Also, unlike humans, they are not capable
of abstract thought, so they cannot use past experience to discern the expected outcome and troubleshoot problems.

Lesson 2: The Structure of HTML Documents

Page Structure

HTML document structure can be described as a tree because all elements are nested within their related parent code. This
video and the following video do a great job of illustrating this concept.

Stylization of Web Pages

CSS, the form to HTML's function, is the code used to stylize web pages. By using CSS to put content in to boxes, web
pages are easily presented in an appealing format. This video gives an easy-to-understand breakdown of the modular
nature of CSS design.

Text Editors, Codepen and Scratchpad

The use of text editors, like Sublime Text can make managing large projects much easier as they are stored on your own
computer rather than on the Web. Codepen is a great tool when you wish to collaborate with others or simply share your
code with other members of the programming community. Scratchpad renders your HTML immediately, which is
useful if you desire instant feedback.

Lesson 3: Adding CSS Style and HTML Structure

Adding Style

As mentioned, we use CSS to add style to HTML documents. CSS can either be written in to an HTML file directly
by using the style or by linking to a separate CSS file by using the link tag. This video provides a little more information
on adding CSS to an HTML document by linking to a separate document.

Cascading Style Sheets

In CSS, the most specific rules are applied to every element. This is known as inheritance, and it allows the programmer
to aviod repetitive coding as all rules are cascaded. This Wikipedia entry explains the concept in depth and provides usable examples.

Selectors

Selectors are used to define which elements of a page are affected by a given style. Class selectors are the most useful
as they work in all browsers and allow you to call multiple classes and assign the same style. This article gives an
in-depth description on the uses of CSS Selectors. The Mozilla Developer Network has a comprehensive list of CSS
attributes and values in a linked document here.
